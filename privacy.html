Privacy Policy

Earl OS · Last updated May 2026

Earl OS (os.earlneal.com) is a personal productivity dashboard built for and used exclusively by Earl Neal. This policy describes how the application handles data.

Who uses this app

Earl OS is a single-user, private application. It is not a public product and is not available for general sign-up. Access is restricted to the account owner.

Google account data

When you sign in with Google, Earl OS requests the following permissions:

  • gmail.readonly
    Read-only access to Gmail messages. Used to display a summary of recent emails in the dashboard so the account owner can monitor their inbox at a glance without leaving the app. No email content is stored on our servers.
  • calendar.readonly
    Read-only access to Google Calendar events. Used to display upcoming events on the dashboard and to auto-populate show journal fields (load-in times, show times, etc.) from calendar event details. No calendar data is stored on our servers.
  • drive.file
    Ability to create and manage files in Google Drive that were created by Earl OS. Used to export daily and show journals as HTML documents to the user's Drive for archiving and sharing. Only files created by Earl OS are accessible; no other Drive files are read.

Google account data accessed through these permissions is used solely to render information in the Earl OS dashboard and is never sold, shared with third parties, or used for advertising.

Data we store

Earl OS stores the following data in a private Supabase (PostgreSQL) database, accessible only to the account owner:

  • Profile information (name, title, status, current event/venue)
  • Task lists and daily task completion state
  • Personal and show journal entries
  • Quick notes
  • Show dates and app settings (theme, notification preferences)

This data is protected by row-level security policies that ensure only the authenticated account owner can read or write their own data.

Data we do not store

  • Gmail message content or metadata
  • Google Calendar event content
  • Google Drive file content (files are written directly to Drive and not retained by Earl OS)
  • Payment information
  • Location data

Third-party services

  • Google APIs — Sign-in and read access to Gmail, Calendar, and Drive as described above. Subject to Google's Privacy Policy.
  • Supabase — Database and authentication infrastructure. Data is stored in a private project. Subject to Supabase's Privacy Policy.
  • Vercel — Static file hosting. No user data is processed by Vercel beyond standard web server logs. Subject to Vercel's Privacy Policy.

Data retention

Data stored in the Supabase database is retained indefinitely until manually deleted by the account owner. Journal entries and task history are intended as a personal archive. No automated deletion schedule is applied.

Security

All data in transit is encrypted via HTTPS. Database access is protected by Supabase row-level security. Google OAuth tokens are managed by Supabase Auth and are not stored in application code or logs.

Your rights

As the sole user, you have full control over your data. You can delete individual entries within the app, or request complete data deletion by removing your account from the Supabase project dashboard.

To revoke Google permissions, visit myaccount.google.com/permissions and remove Earl OS.

Changes to this policy

This policy may be updated to reflect changes in the app's functionality. The "last updated" date at the top of this page will reflect any revisions.
